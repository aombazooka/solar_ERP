# ✅ SolarSell — เช็คลิสต์ก่อน Deploy (Pre-Deployment Checklist)

> ใช้ตรวจสอบให้ครบก่อนนำระบบขึ้นใช้งานจริง (production)
> ทำตามลำดับ — ข้อที่มี ⚠️ คือ "ห้ามข้าม" (ความปลอดภัย/ข้อมูลเสียหาย)

---

## 1. 🔐 ความปลอดภัยฐานข้อมูล (สำคัญสุด)

- [ ] ⚠️ **ตั้งรหัสผ่าน MariaDB `root`** (XAMPP default รหัสว่าง) — รัน [`db/security_setup.sql`](db/security_setup.sql)
- [ ] ⚠️ **สร้าง user เฉพาะแอป** `solarsell_app` (สิทธิ์เฉพาะ DB `solarsell`, ไม่มี DROP/GRANT)
- [ ] ⚠️ แก้ `config/config.php` ให้ใช้ `solarsell_app` (ไม่ใช่ root)
- [ ] ตรวจ `config/config.php` **อยู่ใน `.gitignore`** (ห้าม commit รหัสจริง)
- [ ] ปิด `phpMyAdmin` ไม่ให้เข้าถึงจากอินเทอร์เน็ต (เฉพาะ localhost)

## 2. 👤 บัญชีผู้ใช้ & รหัสผ่าน

- [ ] ⚠️ **เปลี่ยนรหัสผ่านบัญชีทดสอบทั้งหมด** (admin/sales/hr@solarsell.local — default = `password`)
- [ ] ลบ/ปิดใช้งานบัญชีทดสอบที่ไม่ใช้ (เมนู ผู้ใช้งาน)
- [ ] สร้างบัญชีจริงของผู้ใช้แต่ละแผนก + กำหนด role ให้ถูก (admin/sales/installer/finance/hr)
- [ ] ผูกบัญชี user กับข้อมูลพนักงาน (employees.user_id) เพื่อให้ใช้ self-service ได้

## 3. ⚙️ ตั้งค่าแอป

- [ ] ⚠️ ตั้ง `config/config.php` → `'env' => 'production'` และ `'debug' => false` (ปิดการโชว์ error)
- [ ] ตั้ง `'base_url'` ให้ตรงกับ path จริงบน server
- [ ] ตั้ง `'timezone' => 'Asia/Bangkok'`
- [ ] **กรอกข้อมูลบริษัทจริง** (เมนู ตั้งค่าบริษัท) — ชื่อ/ที่อยู่/เลขภาษี → ขึ้นบนเอกสาร
- [ ] ตรวจ session: `httponly` + `samesite=Lax` (ตั้งไว้แล้วใน bootstrap)

## 4. 🗄️ ฐานข้อมูล & ข้อมูลตั้งต้น

- [ ] Import schema + migration ครบทุกไฟล์ตามลำดับ (ดูใน `README.md`)
- [ ] ตรวจ charset = `utf8mb4`, engine = `InnoDB` ทุกตาราง
- [ ] ตั้งค่า **ผังบัญชี / ประเภทวันลา / โควต้า / แผนก-ทีม** ให้ตรงกับองค์กรจริง
- [ ] ตั้ง **จุดสั่งซื้อซ้ำ (reorder_level)** ของสินค้าให้เหมาะสม (มีผลกับแจ้งเตือนสต็อก)
- [ ] ลบข้อมูลตัวอย่าง (ลูกค้า/สินค้า/ใบเสนอราคา demo) ออกถ้าไม่ต้องการ

## 5. 📁 ไฟล์ & สิทธิ์ระบบไฟล์

- [ ] โฟลเดอร์ `storage/uploads/` และ `storage/logs/` **เขียนได้** (web server user)
- [ ] ⚠️ ตรวจ `.htaccess` ใน `config/`, `app/`, `db/`, `storage/`, `tests/` ทำงาน (block การเข้าถึงตรง)
  - บน **Apache**: ต้องเปิด `AllowOverride All` ใน config
  - บน **Nginx**: ต้องเพิ่ม `location` deny เอง (`.htaccess` ไม่ทำงาน)
- [ ] ตรวจ `storage/uploads/checkins/` (รูปเช็คอิน PDPA) เข้าถึงตรงไม่ได้

## 6. 🌐 Server & สภาพแวดล้อม (Production)

- [ ] ⚠️ **เปิด HTTPS** (จำเป็นสำหรับกล้องเช็คอิน `getUserMedia` + GPS + ความปลอดภัย cookie)
- [ ] PHP **8.2+** + extensions: `pdo_mysql`, `mbstring`, `fileinfo` (mime_content_type)
- [ ] ตั้ง `upload_max_filesize` / `post_max_size` ≥ 8MB (รองรับรูปเช็คอิน)
- [ ] ทดสอบบน staging ที่ใกล้ server จริง (Linux) ก่อน — XAMPP ซ่อนปัญหา path/case-sensitive
- [ ] ตั้ง backup ฐานข้อมูลอัตโนมัติ (mysqldump รายวัน) + ทดสอบ restore

## 7. 🧪 ทดสอบก่อนเปิดใช้

- [ ] รันชุดทดสอบ: `php tests/run.php` → ต้องผ่าน **17/17**
- [ ] ทดสอบ login แต่ละ role → เห็นเมนูถูกต้อง (RBAC)
- [ ] ทดสอบกระแสงานหลัก end-to-end:
  - [ ] ขาย: ใบเสนอราคา → ใบสั่งขาย → ส่งของ(ตัดสต็อก) → ออกบิล → รับเงิน
  - [ ] ซื้อ: ใบสั่งซื้อ → รับเข้า(เพิ่มสต็อก+AP) → จ่ายเจ้าหนี้(+WHT)
  - [ ] HR: ตอกบัตร(GPS+รูป) → ยื่นลา/OT → อนุมัติ → ปิดงวดเงินเดือน → สลิป
- [ ] ทดสอบพิมพ์เอกสาร (PDF) → ข้อมูลบริษัทจริงขึ้นถูกต้อง
- [ ] ทดสอบบนมือถือ (responsive) + กล้อง/GPS เช็คอินหน้างาน

## 8. 📜 PDPA & กฎหมาย

- [ ] ขอความยินยอม (consent) ก่อนเก็บรูป/พิกัด GPS ของพนักงาน
- [ ] กำหนดอายุการเก็บข้อมูล (รูปเช็คอิน) + นโยบายลบ
- [ ] จำกัดสิทธิ์เข้าถึงข้อมูลส่วนบุคคล (เงินเดือน/รูป/พิกัด) เฉพาะผู้เกี่ยวข้อง
- [ ] ตรวจรูปแบบใบกำกับภาษี/หัก ณ ที่จ่าย ให้ตรงข้อกำหนดสรรพากร

## 9. 🔄 หลัง Go-Live

- [ ] เฝ้าดู `storage/logs/php-error.log` ช่วงแรก
- [ ] ตรวจ Audit Log (เมนู บันทึกการใช้งาน) ว่าบันทึกครบ
- [ ] commit โค้ดขึ้น Git + tag เวอร์ชัน (อย่า commit `config.php`)
- [ ] เตรียมแผน rollback (backup DB ก่อน deploy ทุกครั้ง)

---

## 🧩 ส่วนที่ยังไม่ได้ทำ (ต่อยอดภายหลัง — ไม่บล็อกการ deploy)

- หนังสือรับรองหัก ณ ที่จ่าย (50 ทวิ) แบบฟอร์มราชการ
- โอนย้ายสต็อกข้ามคลัง (ต้องรองรับหลายคลังก่อน)
- รับเข้าบางส่วนตาม PO (ตอนนี้รับครบทีเดียว)
- สมุดรายวันแบบ auto-post จากทุกธุรกรรม (ตอนนี้มีสมุดรายวันแบบ manual)
- โควต้าวันลาแบบ pro-rate ตามอายุงาน
- 2FA / rate-limit การ login
